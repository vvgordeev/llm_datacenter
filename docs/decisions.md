# Архитектурные решения (ADR)

Краткие записи «почему так». Формат: контекст → решение → последствия. **Новые решения дописывать сверху.**

## ADR-0: Формат этого журнала
Контекст: нужен лёгкий способ фиксировать крупные/неочевидные решения, чтобы они не терялись и не переоткрывались.
Решение: каждое значимое архитектурное решение — короткая запись здесь (3–6 строк): контекст, что решили, последствия/цена. Новые — сверху. Мелкие детали реализации сюда не пишем — им место в `docs/features/*`.
Последствия: история решений живёт рядом с кодом; ревью новых ADR — часть обычного PR.
