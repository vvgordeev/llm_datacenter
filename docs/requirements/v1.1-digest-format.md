# v1.1 — формат конспектов и минимальный объём экспертной секции

**Статус:** accepted · **Дата:** 2026-07-09

## Исходные требования (от пользователя)

1. В ежедневной сборке — **не меньше 5** статей/видео экспертов.
2. Страница конспекта неудобна в текущем виде: вместо перечисления «ключевых тезисов» — **связный текст, кратко рассказывающий суть** материала.
3. Если статья/видео посвящена какому-то **методу** — метод обязательно раскрыть и описать.
4. **Временные метки — прямо по тексту конспекта**, а не отдельным списком.
5. В целом страница конспекта — именно текст, а не искусственно созданные пункты.
6. (Инфраструктурное) youtube.com и googlevideo.com добавлены в trusted domains облачного окружения — кадры из видео должны извлекаться и в облачных прогонах.

## Выявленные конфликты с текущей системой

- Прежний формат конспект-страниц (нумерованные тезисы + отдельная таблица таймкодов + отдельная галерея кадров) заменяется нарративным. Страницы, выпущенные до 2026-07-09, не переделываются задним числом (архив неизменен, инвариант 1); образцом формата служат страницы за 2026-07-09.
- `limits` в sources.yaml задавали только максимумы; вводится минимум суммарного объёма экспертной секции.

## Принятые решения

- `limits.min_expert_total: 5` — если после прохода по экспертам из конфига материалов меньше, скилл расширяет поиск (другие заметные эксперты/материалы недели) до достижения минимума; максимумы подняты до 4+4.
- Формат страницы конспекта: 3–6 абзацев связного текста с подзаголовками по смыслу; раскрытие метода — обязательная часть текста (что за метод, из каких шагов состоит, когда применять); таймкоды видео — инлайн-ссылки в тексте (`[MM:SS]`); кадры — в тексте рядом с фрагментом, к которому относятся, с подписью.
- Кадры в облаке: пробовать извлечение всегда (yt-dlp/imageio-ffmpeg из pip); фиксировать результат в run_log.
