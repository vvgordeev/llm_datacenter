# v1.0 — baseline: ежедневный дашборд новостей LLM

**Статус:** accepted · **Дата:** 2026-07-04

## Исходные требования (от пользователя)

1. Ежедневная сводка-дашборд о новых функциях и подходах к использованию LLM: анонсы из официальных блогов ведущих моделей + реакции сообщества, топовых блогеров и экспертов.
2. По каждому нововведению: краткая сводка, разные мнения, кейсы применения — всё со ссылками.
3. Главное — ранжирование по **масштабу инновации**: насколько сильно влияет на текущие подходы к использованию LLM или порождает новые.
4. Управление источниками: пользователь помечает, по каким моделям собирать сводку.
5. Секция экспертов: популярные видео и статьи лидеров мнений с полноценными конспектами:
   - статьи — отдельная страница: ключевые тезисы, схемы с пояснениями;
   - видео — конспект, тезисы, таймкоды-ссылки, кадры значимых демонстраций (через профильную модель, например Gemini).
6. Обновление раз в день в 8:00 GMT+3; архив дашбордов за предыдущие дни сохраняется.
7. Срезы за неделю и месяц — не всё подряд, а приоритизированное самое важное.
8. Стильный футуристичный дизайн, тёмная/светлая тема.
9. Переключатель языка RU/EN на дашборде (уточнено при обсуждении).

## Выявленные конфликты с текущей системой

Нет — greenfield, первый инкремент.

## Принятые решения по открытым вопросам

- **Хостинг:** GitHub Pages из папки `docs/` этого репозитория.
- **Запуск:** облачная routine Claude Code (cron `0 5 * * *` UTC = 8:00 GMT+3) + локальный скилл `/digest` для ручного пересбора.
- **Язык:** контент хранится двуязычно (`{ru, en}` для всех текстовых полей), переключатель на дашборде.
- **Видео:** Gemini API (анализирует YouTube напрямую); кадры — yt-dlp + ffmpeg по таймкодам от Gemini; fallback без кадров, если инструменты недоступны.
- **Недельный/месячный срезы:** считаются на клиенте из дневных JSON (топ по impact_score, дедуп по cluster_id), ничего не пересобирается.
- **Управление источниками:** правка `config/sources.yaml` (сбор) + фильтры отображения на дашборде (localStorage).

## Допущения

- Бесплатного тира Gemini хватает на 2–3 видео в день (лимит в конфиге).
- X/Twitter плохо индексируется — реакции в основном из HN/Reddit/блогов.
- Кадры видео: jpg среднего качества, ≤6 на видео, чтобы не раздувать репозиторий.

## Техническое задание

- **Данные:** `data/daily/<YYYY-MM-DD>.json` (см. схему в `.claude/skills/digest/SKILL.md`), `data/index.json`, `data/state.json` (дедуп, rolling 60 дней). Копия для сайта — `docs/data/`.
- **Пайплайн (скилл `/digest`):** чтение sources.yaml → сбор (блоги + community за ~36 ч) → дедуп → обогащение (сводка, мнения, кейсы) → ранжирование по рубрике impact_score 1–10 → конспекты (статьи: WebFetch; видео: `scripts/gemini_video.py` + `scripts/extract_frames.py`) → `scripts/build_site.py` → commit/push.
- **UI (`docs/`):** SPA без сборки; виды Сегодня/дата/Неделя/Месяц; карточки по убыванию impact_score с бейджем масштаба; раскрытие сводка→мнения→кейсы; секция экспертов с конспект-страницами (`docs/notes/`); фильтры по моделям; RU/EN; тёмная/светлая тема (CSS variables).
- **Тесты:** unit — чистая логика (агрегация срезов, парсинг таймкодов, build_site); integration — вызовы Gemini/yt-dlp (вручную).
- **Доки:** `docs/features/dashboard.md`, `docs/features/pipeline.md`; инварианты — в `docs/architecture.md`.

## Фазы реализации

1. Документация (этот инкремент, architecture, ADR) + `config/sources.yaml`.
2. Макет дашборда с демо-данными (флаг `demo` + баннер) и примерами конспектов.
3. Скрипты пайплайна + unit-тесты.
4. Скилл `/digest`, тестовый прогон локально за текущую дату.
5. GitHub: репозиторий, Pages (main → /docs).
6. Облачная routine (05:00 UTC) + секрет GEMINI_API_KEY.
