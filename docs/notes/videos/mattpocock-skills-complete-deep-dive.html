Matt Pocock shipped the long-promised complete walkthrough of his Claude Code / Codex skills repository — the one that became the most-installed personal skills repo in the ecosystem (160K stars, 7.5m downloads) without ever having a single tutorial to point to. The video's starting point [00:00] is a contrast between "vibe coding" (chaotic, structureless code generation) and the disciplined engineering practice the repo's skills are built to enforce.

Четыре поломки, которые чинит набор скилловThe four failure modes the toolkit fixes

Покок раскладывает мотивацию всего репозитория на четыре повторяющиеся проблемы агентной разработки [01:09]: агент неверно считывает намерение задачи, выдаёт избыточный (multi verbose) ответ вместо нужного, пишет код с багами и постепенно разрушает архитектуру кодовой базы, если её никто явно не защищает. Дальше видео разбито на две главы скиллов, каждая нацелена на свою половину этих проблем. Pocock breaks the whole repo's motivation into four recurring failures of agentic development [01:09]: the agent misreads task intent, produces an overly verbose response instead of the needed one, writes buggy code, and gradually erodes the codebase's architecture if nothing explicitly guards it. The rest of the video is split into two chapters of skills, each aimed at one half of these failures.

Глава 1 — инженерные скиллы: от согласования модели до TDDChapter 1 — engineering skills: from model alignment to TDD

Первая глава [01:50] посвящена скиллам, которые заставляют агента проектировать и писать код методично. /grill-with-docs [03:06] заставляет агента сперва согласовать доменную модель с человеком, а не сразу писать реализацию. /implement [07:40] — ядро всего конвейера: разработка через тесты (TDD) на заранее согласованных «швах» кода, с обязательным код-ревью перед коммитом. А скилл /tdd [11:27] закрепляет мысль отдельно: агенту нужны короткие, частые циклы обратной связи (написал тест → увидел падение → починил), иначе он накапливает ошибки незаметно для себя между шагами. The first chapter [01:50] covers the skills that force the agent to design and write code methodically. /grill-with-docs [03:06] makes the agent align on a domain model with the human first, rather than jumping straight to implementation. /implement [07:40] is the core of the whole pipeline: test-driven development at pre-agreed code "seams," with a mandatory code review before commit. The /tdd skill [11:27] drives the same point home on its own: the agent needs short, frequent feedback loops (write a test → watch it fail → fix it), or it silently accumulates errors between steps.

Глава 2 — продуктивность: прояснение решений и передача контекстаChapter 2 — productivity: clarifying decisions and handing off context

Вторая глава [14:41] — не про написание кода как такового, а про мышление вокруг него: /grill-me помогает прояснить архитектурное решение до того, как оно закреплено в реализации (тот же принцип «сначала вопросы, потом код», что и в /grill-with-docs, но применённый к произвольным решениям, а не только к доменной модели), а /handoff решает конкретную боль долгих агентных сессий — потерю контекста между ними: скилл упаковывает состояние работы так, чтобы следующая сессия не начинала с нуля. Покок завершает ролик демонстрацией того, как все скиллы стыкуются друг с другом в единый конвейер [18:51] — вывод в том, что ценность не в отдельных файлах, а именно в композиции: каждый скилл занимает свою фазу цикла «идея → спецификация → тикеты → реализация → ревью», и вместе они превращают агента из непредсказуемого соавтора в дисциплинированного исполнителя процесса. The second chapter [14:41] isn't about writing code per se, but about the thinking around it: /grill-me helps clarify an architectural decision before it's locked into an implementation (the same "questions first, code later" principle as /grill-with-docs, but applied to arbitrary decisions rather than just the domain model), while /handoff solves a specific pain point of long agent sessions — losing context between them: the skill packages the state of the work so the next session doesn't start from zero. Pocock closes the video by showing how all the skills snap together into a single pipeline [18:51] — the takeaway being that the value isn't in the individual files but in the composition: each skill owns its phase of the "idea → spec → tickets → implementation → review" cycle, and together they turn the agent from an unpredictable co-writer into a disciplined process executor.
