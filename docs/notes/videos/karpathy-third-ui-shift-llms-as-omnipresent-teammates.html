Andrej Karpathy describes what he calls the "third major shift" in how people interact with LLMs [00:10]. The starting point is his own tweet, read out and unpacked in the video [00:50]: the LLM interface evolves from a website to a desktop app and then on to a persistently present team member.

Claude 1.0 → 2.0 → 3.0: от сайта до вездесущего агентаClaude 1.0 → 2.0 → 3.0: from website to omnipresent agent

Карпати раскладывает эволюцию на три поколения [02:03]: 1.0 — Claude как веб-сайт, разговор в отдельной вкладке браузера; 2.0 — Claude как настольное приложение, глубже встроенное в рабочий стол, но всё ещё отдельная программа, которую нужно открыть; 3.0 — Claude как «вездесущий» участник команды: не приложение, которое запускают, а постоянно работающая, асинхронная сущность, подключённая сразу ко всем системам компании и данным, действующая рядом с людьми как коллега, а не как инструмент под рукой. Karpathy lays out the evolution across three generations [02:03]: 1.0 — Claude as a website, a conversation in its own browser tab; 2.0 — Claude as a desktop app, more deeply embedded on the desktop but still a separate program you have to open; 3.0 — Claude as an "omnipresent" team member: not an app you launch, but a persistent, asynchronous entity wired into all of a company's systems and data at once, acting alongside people like a colleague rather than a tool on hand.

Доказательство на собственном опыте: 65% кода продуктовой командыProof from Anthropic's own use: 65% of the product team's code

Чтобы показать, что это не спекуляция, Карпати ссылается на внутренний опыт Anthropic: похожая на Claude Tag сущность в Slack уже генерирует 65% кода продуктовой команды компании [03:17]. Дальше — живая демонстрация Claude Tag в Slack: агент упоминается в канале, забирает контекст обсуждения, ставит задачу и отчитывается о прогрессе прямо в треде, не требуя переключения в отдельное приложение [03:51]. Более широкое видение, которое он описывает следом, — облачный ИИ, подключённый сразу ко всем внутренним системам компании: базам данных, трекерам задач, документации, — так что агенту не нужно объяснять контекст заново в каждом разговоре [04:02]. To show this isn't speculation, Karpathy points to Anthropic's own internal experience: a Claude-Tag-like entity in Slack already generates 65% of the company's product-team code [03:17]. He follows with a live demo of Claude Tag in Slack: the agent is tagged in a channel, picks up the discussion's context, takes on a task, and reports progress right in the thread, with no need to switch to a separate app [03:51]. The broader vision he describes next is cloud AI hooked up to all of a company's internal systems at once — databases, issue trackers, documentation — so the agent never has to be re-briefed on context in every conversation [04:02].

Личный вездесущий агент: пошаговая сборкаA personal omnipresent agent: a step-by-step build

Оставшуюся часть видео Карпати переносит идею с уровня компании на личный уровень: показывает, как собрать собственного «вездесущего» агента в Telegram, подключённого к личным сервисам и приложениям [08:02]. Пошаговая демонстрация разворачивания — регистрация бота через BotFather, размещение на AWS и подключение инструментов через Composio (сервис, дающий агенту готовые интеграции с внешними API без ручной настройки OAuth для каждого сервиса отдельно) [09:39]. Мысль в том, что тот же паттерн, что работает для команды из Slack-бота на 65% кода, воспроизводим и для одного человека — только мессенджер и набор инструментов свои. For the rest of the video, Karpathy moves the idea from company scale down to personal scale: he shows how to build your own "omnipresent" agent in Telegram, wired into personal services and apps [08:02]. The step-by-step deployment covers registering a bot via BotFather, hosting it on AWS, and wiring up tools through Composio (a service that gives the agent ready-made integrations with external APIs, without hand-rolling OAuth setup for each service separately) [09:39]. The point is that the same pattern behind a team's Slack bot writing 65% of its code reproduces at the scale of one person — just with a different messenger and tool set.
