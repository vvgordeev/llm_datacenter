Доверить агенту собственное суждение вместо пошаговых правил Let agents use their own judgement instead of dictating steps

Оригинал статьи ↗Original article ↗

Ключевые тезисыKey theses

  1. Пошаговые правила не масштабируются. Инструкция вида «используй автотесты только для крупных фич, а для мелких правок правописания — не запускай» быстро набирает исключения и требует постоянной ручной доработки. Step-by-step rules don't scale. An instruction like "only run automated tests for larger features, skip them for small copy edits" quickly accumulates exceptions and needs constant manual upkeep.
  2. Альтернатива — явно передать решение модели. Вместо перечисления случаев автор пишет в системный промпт: «сама решай, когда писать тесты» — и модель справляется с классификацией лучше набора жёстких правил. The alternative is to explicitly hand the decision to the model. Instead of enumerating cases, the author's system prompt just says "use your judgement about when to write tests" — the model classifies these cases better than a fixed rule set.
  3. Тот же приём — для выбора модели субагента. Реальный промпт из статьи: «For all coding tasks use your judgement to decide an appropriate lower power model and run that in a subagent» — модель сама решает, когда мелкую задачу можно делегировать более дешёвой модели. Same trick for picking a subagent's model. The actual prompt from the post: "For all coding tasks use your judgement to decide an appropriate lower power model and run that in a subagent" — the model itself decides when a small task can be delegated to a cheaper model.
  4. Результат по опыту автора: объём выполненной за сессию работы вырос, а лимит на дорогую модель (Fable) расходуется медленнее — экономия достигается без ручной маршрутизации задач. Reported result: more work gets done per session, and the expensive-model (Fable) allowance shrinks more slowly — the savings come without manual task routing.

Почему это важноWhy it matters

Приём стыкуется с общим сдвигом недели: чем меньше в промпте жёстко прописанных процедур и чем больше модели передаётся право решать «как», тем лучше она использует контекст конкретной задачи — и тем дешевле обходится делегирование субагентам. Это дополняет тему автономности в новостях этой сводки (перезапуск Fable/Mythos, разгон инференса GPT-5.6): скорость и доступ к модели становятся менее ценными сами по себе, если промпт не даёт модели свободу решать. The trick lines up with this week's broader shift: the fewer hardcoded procedures in a prompt, and the more the model is trusted to decide "how", the better it can use task-specific context — and the cheaper subagent delegation becomes. It complements this digest's autonomy thread (the Fable/Mythos relaunch, GPT-5.6's inference speed-up): raw model speed and access matter less if the prompt doesn't give the model room to decide.
