Скиллы вместо промптов: как переиспользовать экспертизу агентов Skills over prompts: how to make agent expertise reusable

Оригинал статьи ↗Original article ↗

Ключевые тезисыKey theses

  1. Длинные системные промпты не масштабируются. Они дублируются между проектами, устаревают и не версионируются — знание «как мы работаем» живёт в чатах. Long system prompts don't scale. They get duplicated across projects, go stale and aren't versioned — the "how we work" knowledge lives in chats.
  2. Скилл = инструкция + скрипты + референсы. Упакованная единица экспертизы, которая подключается к агенту по требованию и версионируется как код. A skill = instructions + scripts + references. A packaged unit of expertise, attached to an agent on demand and versioned like code.
  3. Прогрессивное раскрытие — агент читает короткое описание, а полные материалы загружает только когда скилл реально нужен: контекст не раздувается. Progressive disclosure — the agent reads a short description and loads full materials only when the skill is actually needed: the context doesn't bloat.
  4. Командный уровень: общие скиллы через marketplace превращают личные приёмы в стандарты команды. Team level: shared skills via a marketplace turn personal tricks into team standards.
