Разговор с Кэт и Тариком из команды Claude Code A Fireside Chat with Cat and Thariq from the Claude Code team

Оригинал заметки с аннотированной расшифровкой ↗Original post with annotated transcript ↗

На AI Engineer World's Fair Саймон Уиллисон вёл беседу с Кэт Ву и Тариком Шихипаром из команды Claude Code в Anthropic, а затем опубликовал аннотированную расшифровку с дополнительными ссылками. Разговор — не анонс новой функции, а редкий взгляд изнутри на то, как сама команда, которая делает главный агентный инструмент Anthropic, использует его для разработки самого себя, и какими практическими приёмами она держит продукт управляемым. At the AI Engineer World's Fair, Simon Willison sat down with Cat Wu and Thariq Shihipar from Anthropic's Claude Code team, then published an annotated transcript with extra links. The conversation isn't a feature announcement — it's a rare inside look at how the team behind Anthropic's flagship agentic tool uses that same tool to build itself, and the practical habits that keep the product manageable.

Внутренний Slack-агент решает большинство PRAn internal Slack agent lands most PRs

Самая конкретная цифра разговора: Claude Tag — интеграция Claude с коллаборативным чатом команды в Slack — сейчас закрывает 65% продуктовых инженерных pull request'ов внутри самой команды Claude Code. Это не демо и не отдельный эксперимент, а основной канал, через который инженеры формулируют задачу прямо в рабочем чате, а агент доводит её до готового PR. Доля настолько высока, что авторы явно рассматривают Slack как полноценную рабочую поверхность для агентной разработки, а не вспомогательный канал уведомлений. The single most concrete number in the conversation: Claude Tag — Claude's integration into the team's collaborative Slack chat — now closes 65% of the Claude Code team's own product-engineering pull requests. This isn't a demo or a side experiment; it's the primary channel through which engineers state a task right inside a working chat and the agent carries it through to a finished PR. The share is high enough that the team clearly treats Slack as a first-class surface for agentic development, not just a notification channel.

Фичи сначала испытывают на самих себеFeatures get dogfooded before anyone else sees them

Кэт и Тарик описывают процесс поставки фич как ступенчатый: новая возможность Claude Code сначала выкатывается только сотрудникам Anthropic, и только если она демонстрирует устойчивое удержание (retention) именно в этой внутренней когорте, её начинают готовить к внешнему релизу. Это отличается от классического A/B-тестирования на случайной выборке пользователей: здесь сигналом служит поведение людей, которые сами разрабатывают инструмент и знают его сильные и слабые стороны лучше кого-либо, — то есть планка для «фича прижилась» заведомо выше, чем средняя реакция широкой аудитории. Cat and Thariq describe feature delivery as staged: a new Claude Code capability first ships to Anthropic employees only, and it only gets prepared for external release if it shows durable retention within that internal cohort. That's different from classic A/B testing on a random user sample — the signal here comes from the behavior of people who build the tool themselves and know its strengths and weaknesses better than anyone, which sets a deliberately higher bar than an average reaction from a broad audience.

Ручной ревью — для ядра, автоматический — для «внешних слоёв»Manual review for the core, automated review for the "outer layers"

На вопрос о качестве кода, который производят агенты, ответ был не «мы всё автоматизировали» и не «мы всё проверяем руками», а разделение по риску. Критичные изменения — то, что напрямую влияет на поведение агента или на безопасность, — по-прежнему проходят ручной ревью человеком. А для «внешних слоёв» продукта (менее рискованных частей кодовой базы) команда всё активнее полагается на автоматизированный код-ревью, выполняемый другим агентом. Это конкретный пример того, как разработчики Claude Code сами выстраивают многоуровневую систему доверия к агентному коду вместо универсального правила «всегда проверяй» или «всегда доверяй». Asked about the quality of agent-produced code, the answer wasn't "we automated everything" or "we review everything by hand" — it was a risk-based split. Critical changes — anything that directly affects agent behavior or safety — still go through manual human review. For the product's "outer layers" (lower-risk parts of the codebase), the team increasingly relies on automated code review performed by another agent. It's a concrete example of the Claude Code team building its own tiered trust system for agent-written code, instead of a blanket "always review" or "always trust" rule.

Fable монтирует собственное видео анонсаFable edits its own launch video

Отдельный практический штрих: Тарик рассказал, что модель Fable достаточно компетентна в монтаже видео, и он использовал именно её, чтобы смонтировать видео анонса самой Fable. А системный промпт Claude Code, по словам команды, недавно сократился на 80% — конкретное подтверждение того, что снижение служебного «веса» инструкций перед агентом остаётся приоритетом внутренней разработки, а не разовой оптимизацией. A separate practical detail: Thariq said Fable is competent enough at video editing that he used it to edit Fable's own launch video. And Claude Code's own system prompt, per the team, recently shrank by 80% — concrete confirmation that trimming the instructional "weight" placed in front of the agent remains an ongoing internal priority, not a one-off optimization.
