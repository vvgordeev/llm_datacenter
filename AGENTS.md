# LLM Datacenter — карта для агентов

Ежедневная сводка-дашборд новостей об LLM: анонсы функций ведущих моделей из официальных блогов, реакции сообщества и экспертов, конспекты статей и видео, ранжирование по масштабу инновации. Продакшен: GitHub Pages (корень сайта — папка `docs/`). Требования к продукту: `docs/requirements/` (журнал инкрементов; актуальное состояние системы — в `docs/features/*`). Метод работы над проектом — `CONVENTIONS.md`.

## Стек (версии пинить!)

Python 3.12+ (скрипты пайплайна) · pytest · ruff · vanilla JS/HTML/CSS без сборки (дашборд в `docs/`) · Gemini API (конспекты видео) · yt-dlp + ffmpeg (кадры из видео) · Claude Code skill `/digest` (ежедневный пайплайн сбора)

## Команды

```
python -m http.server 8000 -d docs   # локальный просмотр дашборда
ruff check . && pytest tests/unit    # обязательный минимум перед коммитом
pytest tests/integration             # сетевые/интеграционные тесты (вручную; нужны ключи API)
```

## Грабли окружения

Разработка на Windows 11 (PowerShell). `GEMINI_API_KEY` — только в переменных окружения. Для извлечения кадров из видео нужны `yt-dlp` и `ffmpeg` в PATH. Пути с кириллицей (`C:\Users\v-gor\Проекты\...`) — в скриптах не полагаться на ASCII-пути.

## Карта документации — что читать перед задачей

Сначала всегда: `docs/architecture.md` (слои, инварианты, матрица доступа). Затем файл области:

| Задача касается | Читай |
|---|---|

## Правила работы

1. **Изменил поведение — обнови соответствующий `docs/features/*.md` в том же коммите.** Устаревший док хуже отсутствующего.
2. Не нарушать инварианты из `docs/architecture.md`.
3. Авторизация/доступ — в одном явном слое (сервисы/guards), а не размазана; описать в architecture.md.
4. Новые строки UI — через систему i18n проекта (дашборд двуязычный RU/EN, все текстовые поля данных хранятся парой `{ru, en}`).
5. Крупное архитектурное решение — короткая запись в `docs/decisions.md` (сверху, ADR).
6. **Делегирование субагентам — только через навык `/delegate`** (явный вызов): декомпозиция → оценка выгоды → риски → подтверждение. По своей инициативе работу не делегировать; read-only разведка — можно. Никогда не делегируются: auth/безопасность, инварианты architecture.md, миграции БД.
7. **Задачи и баги** — трекер пока не настроен; фиксировать в `docs/roadmap.md`. При подключении трекера обновить этот пункт (автономным/headless-агентам понадобится API key — интерактивный OAuth в headless не работает).
